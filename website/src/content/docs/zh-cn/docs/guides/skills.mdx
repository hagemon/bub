Bub 的 skills 系统遵循 [Agent Skills](https://agentskills.io/)：一个 skill 是以 `SKILL.md` 为中心的可发现目录，并可包含 `scripts/`、`references/`、`assets/` 等辅助文件。
下面的内容主要说明 Bub 如何加载这些 skill，以及我们推荐的编写方式。

## 最小契约

每个技能目录必须包含一个 `SKILL.md` 文件：

```text
my-skill/
`-- SKILL.md
```

Agent Skills 中常见的可选目录还有：

- `scripts/`：可执行辅助脚本
- `references/`：按需加载的参考文档
- `assets/`：模板和静态资源

Bub 当前的加载器会读取并要求：

- `SKILL.md` 必须以 YAML frontmatter 开头（`--- ... ---`）
- frontmatter 必须包含非空的 `name` 和 `description`
- 目录名必须与 frontmatter 中的 `name` 完全匹配
- `name` 必须匹配正则 `^[a-z0-9]+(?:-[a-z0-9]+)*$`
- `name` 长度必须 `<= 64`
- `description` 长度必须 `<= 1024`
- 如果提供了 `metadata`，必须是 `string -> string` 的映射

## Frontmatter 字段

Bub 当前会读取的字段：

- 必需：`name`、`description`
- 可选（带类型检查）：`metadata`

允许其他额外字段。

## 发现与覆盖

技能从以下三个根目录按优先级顺序发现：

1. 项目级：`.agents/skills`
2. 用户级：`~/.agents/skills`
3. 内置：`src/skills`

如果名称冲突，列表中靠前的根目录胜出。

这是 Bub 在标准 Agent Skills 格式之上的运行时发现行为。

遗留路径说明：`.agent/skills` 仍会被扫描以保持兼容，但 Bub 会发出弃用警告并优先使用 `.agents/skills`。

## 运行时访问

内置命令模式可以检查已发现的技能：

```bash
uv run bub run ",skill name=my-skill"
```

如果技能未找到，`,skill name=<name>` 会返回 `(no such skill)`。

## 编写指南

- 保持 `SKILL.md` 简洁且面向操作
- 保持 metadata 精简且确定性
- 使用小写 kebab-case 命名以保证兼容性
- 详细资料尽量放入 `references/` 或脚本，而不是把 `SKILL.md` 写得过长

## 可选脚本约定

对于 `scripts/*.py`，推荐使用 PEP 723 配合 `uv` 的独立脚本约定：

```python
#!/usr/bin/env -S uv run --script
# /// script
# requires-python = ">=3.12"
# dependencies = []
# ///
```
